2.1 View 层：MainWindow 与数据流相关的状态

成员 / 控件类型或产出在数据流中的作用
viewmodelSimulationViewModelQt 视图与无 Qt ViewModel 的分界；所有业务编排经此对象。
parameter_table（及行收集）表格 → list[dict] 关节行_collect_link_rows 等读出 MDH / 关节名等，送入 build_config(..., link_rows, ...)。

command_editstr自然语言：parse_task_blocks(text)；同时驱动可行性定时器。
task_table表格 ⇄ TaskBlock用户编辑结构化任务：_collect_task_blocks_from_table() → run_task_blocks。
current_resultsdict | None最近一次成套结果：fk_demo、ptp、cartesian_track、active_config（见下文表）。绑定 3D、关节曲线与文本摘要。
current_robotSixDofRobot | None与画布一致的 FK 几何实例：robot_from_config(active_config) 后与 current_results["ptp"]/"cartesian_track" 一起绘制。
current_task_blockslist[TaskBlock]解析后与任务表保持同步；执行多段仿真时送入 ViewModel。
animation_result / animation_frame轨迹快照 + 帧索引播放器按时间片消费与 cartesian_track 同型的数组字段（如 q_rad、position_m）。
feasibility_labelQLabel展示 precheck_task_blocks + summarize_precheck 的可行性摘要字符串。
robot_canvas / joint_canvas / result_box控件消费端：SixDofRobot + current_results 的字典与时间序列字段。

2.4 current_results 顶层键；cartesian_track（与 evaluate_trajectory）

下列与 python/trajectory.py 中 evaluate_trajectory、python/services/trajectory_service.py 的 run_commands 返回值一致；多任务时 ViewModel 在 cartesian_track 上追加 task_blocks。

键说明（消费者）
fk_demoSimulationService.run_fk(config)：静态 FK 预览（轨迹包装路径）。
ptpcartesian_track 的深拷贝（仅 summary.trajectory_type 改为预览语义）；结构与下列「轨迹本体」同源。
cartesian_track轨迹服务输出的主时间序列字典（见下行扩展）。画布、播放器、导出与误差卡片主要读此项。
active_config本次运行使用的 config 快照；重现 SixDofRobot。

cartesian_track 内（在 evaluate_trajectory 基础上由 TrajectoryService 增补）：

  • 阵列与时间：time_s，q_rad / q_deg，position_m，desired_position_m，rpy_deg，transforms
  • summary：机器人名、source、trajectory_type、采样数、时间跨度等
  • metrics：max_position_error_m、mean_position_error_m（来自评测）；外加 ik_failure_count / ik_failure_ratio，max_jacobian_condition、near_singular_count；以及关节步长（max_joint_step_deg 等）与地面/连杆间隙类标量（见源码 _joint_motion_metrics、_environment_motion_metrics）
  • 命令与诊断：precheck，commands / requested_commands（多段时为列表），末尾单段的 command / requested_command，segment_diagnostics
  • 多任务：task_blocks（仅 run_task_blocks）
